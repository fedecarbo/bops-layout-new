# Select UI Block Workflow - Checklist

## Pre-Workflow

- [ ] User has described what they want to build
- [ ] UI blocks library is available at expected location
- [ ] Tech KB catalog.md is accessible

## Step 1: Understand Requirements

- [ ] Analyzed user request for component type
- [ ] Identified category (forms, navigation, layout, etc.)
- [ ] Extracted style preferences (dark/light, simple/complex)
- [ ] Determined scope (single component vs. multi-component)

## Step 2: Load UI Blocks Knowledge

- [ ] Loaded catalog.md from tech-kb
- [ ] Loaded essentials.md for context
- [ ] Confirmed ui-blocks location path

## Step 3: Search Components

- [ ] Searched use case recommendations (if applicable)
- [ ] Searched by category
- [ ] Searched by tags (if specific features mentioned)
- [ ] Found 3-5 matching components
- [ ] Ranked by relevance and simplicity

## Step 4: Present Options

- [ ] Presented 3-5 options with clear descriptions
- [ ] Included location path for each option
- [ ] Listed tags and dependencies
- [ ] Explained when to use each variant
- [ ] Asked user to select or request alternatives

## Step 5: Retrieve Code

- [ ] User selected component
- [ ] Read component file from ui-blocks library
- [ ] Extracted frontmatter metadata
- [ ] Extracted complete TSX code block
- [ ] Identified related/complementary components

## Step 6: Deliver Component

- [ ] Provided component name and location
- [ ] Delivered exact code (unmodified)
- [ ] Listed all dependencies with install command
- [ ] Included usage instructions
- [ ] Noted accessibility features
- [ ] Shared technical notes from frontmatter
- [ ] Suggested related components (if applicable)
- [ ] Offered assistance with integration

## Post-Workflow

- [ ] Component code successfully delivered
- [ ] User understands how to use it
- [ ] Dependencies clearly communicated
- [ ] Related components suggested
- [ ] User ready to integrate or requested additional components

## Quality Checks

- [ ] Component matches user's stated requirements
- [ ] Code is production-ready (from Tailwind UI)
- [ ] All imports and dependencies identified
- [ ] Accessibility features preserved
- [ ] Customization guidance provided
- [ ] Related components enhance solution

## Success Indicators

✅ User selected one of the presented options (didn't need multiple refinements)
✅ Component required minimal/no modification to meet needs
✅ Found match within first search attempt
✅ Suggested related components that user actually needed
✅ Complete solution delivered (not just partial)
